# ========================================
# PASSO-A-PASSO PASSO A PASSO:
# ========================================

1. GERAR SECRET_KEY
   - Abra PowerShell/Terminal
   - Digite: python -c "import secrets; print(secrets.token_hex(32))"
   - Copie o resultado (será uma string longa com números e letras)

2. ADICIONAR VARIÁVEIS NO RAILWAY
   - Acesse: https://railway.app
   - Projeto: battlezone-production
   - Settings > Variables
   - Clique "+ Add Variable"
   - Nome: FLASK_ENV
   - Valor: production
   - Clique "Add"
   
   - Repita para SECRET_KEY (usar o valor gerado no passo 1)

3. HABILITAR POSTGRESQL
   - Em railway.app, seu projeto
   - Vá para: Resources (ou + Add)
   - Procure: PostgreSQL
   - Clique em PostgreSQL para adicionar
   - Aguarde 2-5 minutos
   - DATABASE_URL aparecerá automaticamente

4. REDEPLOY
   - Volte ao VS Code
   - Digite: git push origin main
   - Railway fará novo deploy automaticamente

5. VERIFICAR
   - Aguarde 2-3 minutos
   - Acesse: https://battlezone-production.up.railway.app
   - Deve funcionar agora!

# ========================================
# TROUBLESHOOTING
# ========================================

Se ainda der 502 Bad Gateway:

1. Verifique se tem PostgreSQL habilitado:
   - railway.app > seu projeto
   - Veja se tem um card "PostgreSQL"
   - Se não tem, adicione!

2. Verifique as variáveis:
   - railway.app > seu projeto > Settings > Variables
   - FLASK_ENV DEVE ser: production
   - SECRET_KEY DEVE ter um valor longo
   - DATABASE_URL DEVE aparecer

3. Verifique os logs:
   - railway.app > seu projeto > Deployments
   - Clique no deploy mais recente
   - Veja a aba "Logs"
   - Procure por erro específico

4. Se tudo estiver certo:
   - Force redeploy: git add -A && git commit --allow-empty -m "redeploy" && git push origin main
